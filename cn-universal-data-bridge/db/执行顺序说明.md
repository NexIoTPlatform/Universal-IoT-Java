# 字典数据导入执行顺序

## 执行步骤

请按照以下顺序执行SQL文件：

### 1. 清理现有数据（可选）

如果需要重新导入，请先执行：

```sql
DELETE FROM sys_dict_data WHERE dict_type LIKE 'databridge_%';
DELETE FROM sys_dict_type WHERE dict_type LIKE 'databridge_%';
```

### 2. 按顺序执行SQL文件

1. **01_字典类型.sql** - 创建所有字典类型
2. **02_基础字典数据.sql** - 创建基础字典数据（资源类型、状态等）
3. **03_MySQL配置字段.sql** - MySQL配置字段
4. **04_Kafka配置字段.sql** - Kafka配置字段
5. **05_MQTT配置字段.sql** - MQTT配置字段
6. **06_HTTP配置字段.sql** - HTTP配置字段
7. **07_IoTDB配置字段.sql** - IoTDB配置字段
8. **08_InfluxDB配置字段.sql** - InfluxDB配置字段
9. **09_Elasticsearch配置字段.sql** - Elasticsearch配置字段
10. **10_Redis配置字段.sql** - Redis配置字段
11. **11_云平台配置字段.sql** - 云平台配置字段（阿里云、腾讯云、华为云）

## 验证数据

执行完成后，可以运行以下SQL验证数据：

```sql
-- 检查字典类型
SELECT * FROM sys_dict_type WHERE dict_type LIKE 'databridge_%' ORDER BY dict_type;

-- 检查各资源类型的配置字段数量
SELECT 
    dict_type,
    COUNT(*) as field_count
FROM sys_dict_data 
WHERE dict_type LIKE 'databridge_%_fields' 
GROUP BY dict_type 
ORDER BY dict_type;
```
